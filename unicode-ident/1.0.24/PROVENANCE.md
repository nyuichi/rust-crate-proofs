# unicode-ident 1.0.24 provenance and verification scope

**Verification status: complete-equivalent.**

This source tree is copied from the crate published on crates.io as
`unicode-ident` version `1.0.24`. The published archive has SHA-256 checksum
`e6e4313cd5fcd3dad5cafa179702e2b244f760991f45397d14d4ebf38247da75`.
Its `.cargo_vcs_info.json` records upstream revision
`5b54a632702b5744a1c40ea01c127c0ac0498172`. The generated tables implement
Unicode 17.0.0.

The public runtime API, `no_std` behavior, licenses, compressed tables, and
optimized lookup implementation are preserved. The verification manifest uses
the package name `unicode-ident-verify` while retaining the published library
target name `unicode_ident`; this avoids ambiguity with Creusot's transitive
dependency on another `unicode-ident` version.

## Established contracts

The proof defines exact models for ASCII XID_Start and XID_Continue membership
and for the generated start-trie, continue-trie, and shared-leaf lookup. It
establishes:

- exact ASCII classification for letters, digits, and underscore;
- safe non-ASCII trie indexing and leaf-offset arithmetic for every `char`;
- exact extraction of each of the eight possible bits from a leaf byte;
- exact composition of the trie byte, half-chunk offset, leaf byte, and bit
  position for both XID properties;
- both public functions return their corresponding classification model.

The verification implementation uses division by a selected power of two for
leaf-bit extraction because this Creusot revision does not model the published
`wrapping_shr` spelling. The included exhaustive runtime test checks every
valid Unicode scalar value against the independently generated Unicode 17.0.0
range tables, connecting the published optimized implementation to the same
classification proved for the verification implementation.

## Explicit trusted boundaries

Creusot cannot translate immutable static arrays or a cast from `char` to an
integer in this revision. Four narrow declarations isolate those limitations:

- `char_codepoint`, whose contract preserves the exact scalar value;
- `trie_start_at` and `trie_continue_at`, whose contracts expose the exact
  modeled generated byte and its bound needed for leaf safety;
- `leaf_at`, whose contract exposes the exact modeled generated leaf byte.

The table accessors are the only generated-data boundary. Their removal
condition is Creusot support for immutable static arrays, allowing their bodies
to be proved directly against `TRIE_START`, `TRIE_CONTINUE`, and `LEAF`.
Likewise, `char_codepoint` can be removed when `char`-to-integer casts translate.
The correspondence between the compressed tables and the Unicode 17.0.0 range
data is tested exhaustively rather than assumed from selected examples.
